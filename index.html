Tesla Software Internship

As a Software Development Intern on Tesla's Cell Material Flow Engineering team, I delivered Android app features that track and identify cell materials across Giga Texas. The release deployed to 12 Zebra handheld scanners and let operators dynamically associate or clear RFID tags, giving real-time traceability for 100+ containers each day.

I implemented an MVVM architecture with Kotlin, Java, and Gradle to keep the codebase modular, and generated gRPC clients from .proto files so the scanners could query Tesla internal APIs for material quality data without latency.

I partnered with controls engineers to weave Beckhoff PLC logic into the handheld workflow, then led a gap analysis across 200+ tracking failures to surface issues like sensor misalignment and ambiguous clear logic, informing system updates that improved reliability and reduced material loss incidents.

  • Stack: Kotlin, Java, Gradle, gRPC, Beckhoff PLC integration
  • Scale: 12 scanners in production and 100+ containers tracked per day
  • Analysis: 200+ failures reviewed to improve floor reliability

Mini-DLSS - DLSS-Inspired Temporal Video Super-Resolution

PyTorch / Video Super-Resolution / Temporal Modeling

Built a DLSS-inspired temporal video super-resolution pipeline that reconstructs high-resolution frames from low-resolution inputs using multi-frame context to improve detail and reduce temporal artifacts.

The project focuses on machine learning depth: large-scale dataset prep, temporal model training, quality and latency benchmarking, and model-size vs. performance tradeoff analysis.

Open GitHub Repo

Scale and Training Pipeline

  • Data pipeline scale: 12,824 train sequences, 38,472 temporal windows, and 89,768 HR frames, plus 674 validation sequences.
  • Data quality: 0 alignment mismatches on a 20-sequence local REDS-style validation subset.
  • Training throughput: 2,000 steps in 121.41s (~16.47 steps/s).

Model and Results

  • Model efficiency: temporal model uses 0.716M parameters vs 0.742M for single-frame (~3.5% fewer).
  • Quality gain with longer training: PSNR-Y improved from 29.2989 to 33.5138 (+4.2149 dB), and SSIM-Y improved from 0.8532 to 0.9247 (+0.0715), from 600 to 2,000 steps.
  • Temporal stability: tPSNR reached 37.6004 vs 33.8817 for single-frame (+3.7187 dB).
  • Inference speed (CPU): temporal model at 34.66 ms/frame (28.9 FPS) vs single-frame at 9.105 ms/frame (109.8 FPS).

Evaluation caveat: these metrics are measured on a local REDS-style validation set, not the official REDS benchmark.

UWAFT ECO Car Hackathon - 1st Place

University of Waterloo / MATLAB / Simulink / YOLO

Built the perception-to-control loop that let our autonomous taxi behave like a rules-of-the-road vehicle inside MATLAB/Simulink.

Computer Vision / Stop-Sign Detection (ACF)

Trained an ACF (Aggregated Channel Features) stop-sign detector in MATLAB using ground-truth bounding boxes. The trained model outputs a bounding box and an ACF score showing how strongly a region matches learned stop-sign features. I validated the detector on held-out images and tuned the detection threshold to balance missed detections against false positives.

AI/ML / Traffic-Light Detection (YOLO)

To extend perception beyond stop signs, I trained a YOLO-based traffic-light detector in Python using Google Colab GPUs (CUDA) to classify traffic-light states (red / yellow / green). On validation the model achieved:

  • 88.7% Precision - high correctness of detections
  • 86.2% Recall - strong coverage of actual objects
  • 91.3% mAP@50 - robust overall detection performance

This gave us a modern deep-learning perception component that complements the lightweight ACF approach, evaluated using standard object-detection metrics rather than visual inspection alone.

Controls Integration / Simulink SpeedController

Integrated perception outputs into the Simulink control stack by converting detections into a desired-speed override that feeds the existing SpeedController. Rather than driving motors directly from vision, the perception layer produces a clean stop-or-go decision, and the controller handles smooth deceleration to 0 m/s and resumption to cruise speed.

This kept the system modular: perception answers what is present, while control answers how the vehicle should respond. The result was more predictable vehicle behavior during live demos.

  • Python / YOLO - deep-learning traffic-light classification
  • Google Colab (CUDA) - GPU-accelerated model training

The controller supports continuous and fixed-package modes, pauses when the path is blocked, and triggers a goHome recovery flow if the obstruction persists for more than 30 seconds.
